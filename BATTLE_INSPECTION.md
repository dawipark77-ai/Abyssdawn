## 전체 진행 순서

```
0단계 씬 영속성 (선행 필수)
  ↓

  ↓

  ↓

  ↓
4단계 스킬 검증
  ↓
```

## 0단계 — 씬 전환 데이터 영속성 ⚠️ 선행 필수

> 이게 깨지면 이후 모든 시뮬레이션이 의미 없음. 가장 먼저 확인.

| # | 항목 | 검증 방법 | 상태 |
|---|------|-----------|------|
| 0-1 | 던전 → 전투 씬 전환 시 `staticPartyData` 유지되는가 | 던전에서 인카운터 발생, 전투 씬 진입 후 파티 데이터 확인 | ☐ |
| 0-2 | 전투 종료 → 던전 복귀 시 HP/MP가 올바르게 반영되는가 | 전투에서 피해 입은 뒤 복귀, 던전 씬의 HP 확인 | ☐ |
| 0-3 | `DungeonPersistentData.currentHP/currentMP` 저장/복원이 올바른가 | BattleManager 전투 종료 코드 → DungeonGridPlayer 복원 코드 추적 | ☐ |
| 0-4 | 전투 중 사망 시 씬 전환이 올바르게 처리되는가 | 전원 사망 상황 유도 후 씬 전환 확인 | ☐ |
| 0-5 | 여러 번 인카운터 반복해도 데이터 누적 오염 없는가 | 던전 이동 → 전투 → 복귀 3회 반복 | ☐ |

## 1단계 — 캐릭터 스탯 계산 검증

| # | 항목 | 검증 방법 | 상태 |
|---|------|-----------|------|
| 1-1 | 기본값 × `CharacterClass` 배율이 올바르게 계산되는가 | Inspector에서 직업 변경 후 스탯 수치 확인 | ☐ |
| 1-2 | 장비 장착/해제 시 스탯이 즉시 반영되는가 | 장비 착탈 후 `PlayerStats` 수치 확인 | ☐ |
| 1-3 | 패시브 보너스가 스탯에 반영되는가 | PassiveData 할당 후 스탯 비교 | ☐ |
| 1-4 | `MemoryOfSpecies` 개별 보너스가 스탯에 반영되는가 | 단일 기억 장착 후 수치 확인 | ☐ |
| 1-5 | `MemoryOfSpecies` **3세트** 발동 시 `TraitsOfSpecies` 보너스 적용되는가 ⚠️ | 동종 기억 3개 장착, OnValidate 후 스탯 확인 (런타임에서 누락 가능) | ☐ |
| 1-6 | 레벨 성장치(`CharacterClass` 레벨당 성장)가 올바르게 누적되는가 | 레벨 올린 뒤 HP/Attack 수치 확인 | ☐ |
| 1-7 | `maxHP`, `maxMP` computed property가 전투 시작 시 올바른가 | BattleManager Start() 시점의 HP/MP 값 | ☐ |
| 1-8 | 양손 무기 장착 시 왼손 슬롯 자동 해제 후 스탯 반영 올바른가 | 양손 무기 → 왼손 스탯 제거 확인 | ☐ |

## 2단계 — 턴 구조 검증

| # | 항목 | 검증 방법 | 상태 |
|---|------|-----------|------|
| 2-1 | Agility 순서대로 턴이 돌아가는가 | 고민첩 적 vs 저민첩 아군, 턴 순서 로그 확인 | ☐ |
| 2-2 | 동일 Agility 시 처리 순서가 일관적인가 | Agility 동값 캐릭터로 반복 테스트 | ☐ |
| 2-3 | 아군/적군 교대 턴이 올바르게 동작하는가 | 턴 순서 로그 확인 | ☐ |
| 2-4 | 전열(Slot1~2) / 후열(Slot3~4) 슬롯 구조가 올바른가 | `SlotMask` 비트마스크 동작 확인 | ☐ |
| 2-5 | 스턴 상태에서 행동 불가 처리되는가 | 스턴 부여 후 해당 캐릭터 턴 스킵 확인 | ☐ |
| 2-6 | 캐릭터 사망 시 턴 큐에서 제거되는가 | 전투 중 사망 유도, 이후 턴 흐름 확인 | ☐ |
| 2-7 | 전투 종료 조건 (전원 사망) 이 올바르게 발동되는가 | 아군 전원/적군 전원 사망 상황 유도 | ☐ |
| 2-8 | `battleEnded` 플래그가 중복 처리를 방지하는가 | 동시 사망 엣지케이스 테스트 | ☐ |

## 3단계 — 상태이상(저주/Curse) 검증

> 기준 수치 (현재 SO 설정값 기준으로 변경 가능)

| 상태이상 | DoT 비율 | 지속 턴 | 부여 무기 |
|---------|---------|--------|---------|
| Bleed (Dagger) | 4%/턴 | 3턴 | Crude_Dagger |
| Bleed (Katana) | 4%/턴 | 3턴 | Crude_Katana |
| Poison | 2%/턴 | 10턴 | — |
| Ignite | 6%/턴 | 3턴 | — |
| Stun | — | 1턴 | — |

| # | 항목 | 검증 방법 | 상태 |
|---|------|-----------|------|
| 3-1 | 부여 확률(`applyChance`)이 올바르게 굴려지는가 | SO의 applyChance 값 vs 실제 부여 빈도 비교 | ☐ |
| 3-2 | 물리/마법 DoT가 매 턴 시작 시 처리되는가 | 상태이상 부여 후 턴별 HP 감소 로그 확인 | ☐ |
| 3-3 | 지속 턴 카운트가 매 턴 1씩 올바르게 감소하는가 | 턴 카운트 로그 확인 | ☐ |
| 3-4 | 지속 턴 만료 후 자동 해제되는가 | 3턴 후 Bleed 해제 확인 | ☐ |
| 3-5 | 중첩 처리가 올바른가 (갱신 vs 추가 vs 거부) | 동일 상태이상 2회 연속 부여 테스트 | ☐ |
| 3-6 | 스턴: 턴 종료 후 자동 해제되는가 | 스턴 후 다음 턴 행동 가능 여부 | ☐ |
| 3-7 | 상태이상 아이콘이 `EnemyStats` World-Space UI에 표시되는가 | 부여 후 적 UI 확인 | ☐ |
| 3-8 | `itemIcon` / `flatIcon` 중 올바른 아이콘이 전투 UI에 사용되는가 | flatIcon이 전투 화면에 표시되는지 확인 | ☐ |

## 4단계 — 스킬 검증

| # | 항목 | 검증 방법 | 상태 |
|---|------|-----------|------|
| 4-1 | MP 소모 스킬이 올바르게 MP 차감되는가 | 스킬 사용 전후 MP 수치 확인 | ☐ |
| 4-2 | MP 부족 시 스킬 사용이 차단되는가 | MP 0 상태에서 스킬 버튼 확인 | ☐ |
| 4-3 | 스킬 배율(`multiplier`)이 데미지에 적용되는가 | 기본 공격 데미지 vs 스킬 데미지 비교 | ☐ |
| 4-4 | `allowedCasterSlots` 포지션 제한이 작동하는가 | 후열 전용 스킬을 전열 캐릭터가 사용하려 할 때 | ☐ |
| 4-5 | `allowedTargetSlots` 타겟 제한이 작동하는가 | Front Only 스킬이 후열 적을 타겟 못하는지 | ☐ |
| 4-6 | 다중 타격 스킬 타수가 SO 설정값과 일치하는가 | 로그에서 히트 횟수 확인 | ☐ |
| 4-7 | 듀얼 장착 시 × 0.35~0.45 배율 적용되는가 | 단일 장착 vs 듀얼 데미지 비교 | ☐ |
| 4-8 | 선행 스킬(`prerequisiteSkills`) 미충족 시 스킬 잠금되는가 | 선행 스킬 없이 하위 스킬 학습 시도 | ☐ |
| 4-9 | `DunBreak` 확률이 올바르게 적용되는가 | DunBreak 스킬 반복 사용, 발동 빈도 확인 | ☐ |
| 4-10 | 소비 아이템 사용 시 `ConsumableInventory.UseItem()` 호출 및 수량 차감되는가 | 전투 중 아이템 사용 후 인벤토리 수량 확인 | ☐ |
| 4-11 | `hpRecoveryPercent × maxHP` 힐량 계산이 올바른가 | HP_Potion 사용 후 회복량 수치 확인 | ☐ |

## 5단계 — 특수 시스템 검증 (현재 구현된 것만)

> ⚠️ **Backlash(역효과) 시스템은 미구현** — 이 단계에서 제외. 8단계에서 별도 구현 예정.

| # | 항목 | 검증 방법 | 상태 |
|---|------|-----------|------|
| 5-1 | 방어구 파괴(`ArmorBreak`) 누적 감소가 올바른가 | 타격마다 `현재 방어력 × coefficient` 차감 확인 | ☐ |
| 5-2 | 방어력이 0 이하로 내려가지 않는가 | 방어력 0 상태에서 추가 ArmorBreak 사용 | ☐ |
| 5-3 | ArmorBreak 크리티컬 적용이 올바른가 | 크리티컬 시 ArmorBreak 데미지 1.5× 확인 | ☐ |
| 5-4 | 블록(`Block`) 확률 계산이 올바른가 | 방패 장착 캐릭터의 블록 발동 빈도 확인 | ☐ |
| 5-5 | 블록 발동 시 데미지 감소가 올바른가 | 블록 발동 전후 받는 데미지 비교 | ☐ |
| 5-6 | 방어(`Defend`) 행동 시 `defenceReduction 0.5f` 적용되는가 | 방어 후 받는 피해가 50% 수준인지 확인 | ☐ |
| 5-7 | 명중 계산 `attackerAgility × 2 - targetAgility` 공식이 올바른가 | 고민첩 vs 저민첩 명중률 로그 확인 | ☐ |
| 5-8 | 명중률 최소 10% / 최대 100% 캡이 적용되는가 | 극단적 민첩 차이 케이스 테스트 | ☐ |
| 5-9 | 크리티컬 `luck × 0.05f` 공식이 올바른가 | luck 20 = 100% 크리티컬 상한 확인 | ☐ |
| 5-10 | `BattleRecorder` F8 일시정지/F6~F7 되감기가 동작하는가 | 전투 중 단축키 테스트 | ☐ |

## 6단계 — BalanceSimulator 실행 → 기준값 확정

| # | 항목 | 확인 방법 | 상태 |
|---|------|-----------|------|
| 6-1 | `BalanceSimulator.cs` `[ContextMenu]` 실행 가능한가 | Inspector 우클릭 → Simulate Balance | ☐ |
| 6-2 | 50층 클리어 시뮬레이션 결과 기록 | 승률, 평균 생존 층, 전투당 평균 턴 수 기록 | ☐ |

| 6-4 | 상태이상 없는 순수 물리 파티 vs 상태이상 파티 비교 | 두 조합 시뮬레이션 후 수치 비교 | ☐ |

## 7단계 — UI 연동 검증 (별도 작업 / 로직 점검 완료 후 진행)

> 전투 로직 점검과 분리. 6단계 완료 후 별도로 진행.

| # | 항목 | 검증 방법 | 상태 |
|---|------|-----------|------|
| 7-1 | HP/MP 바가 피해/회복 시 실시간 갱신되는가 | 전투 중 HP 변화 → UI 반영 확인 | ☐ |
| 7-2 | 상태이상 아이콘이 부여/해제 시 올바르게 표시/제거되는가 | Bleed 부여 후 3턴 뒤 아이콘 사라지는지 | ☐ |
| 7-3 | 데미지 숫자(팝업 또는 로그)가 표시되는가 | 공격 후 수치 확인 | ☐ |
| 7-4 | 턴 순서 UI가 현재 액션 캐릭터를 올바르게 표시하는가 | 턴 진행 중 포커스 표시 확인 | ☐ |
| 7-5 | 파티 상태 패널(playerStatusPanel)이 파티원 수에 맞게 조정되는가 | 파티원 1명 / 4명 케이스 비교 | ☐ |
| 7-6 | `StatRow` ◆ 컬러 접두어가 인벤토리 UI에서 올바르게 표시되는가 | 무기 상세 보기 확인 | ☐ |
| 7-7 | `selectedConsumableItem` 이 인벤토리 선택 시 자동으로 설정되는가 ⚠️ 미완성 | 현재 수동 연결 → 자동화 구현 필요 | ☐ |
